# Mithrandir — Development Guide for Claude Code

## What This Project Is

Mithrandir is a personal AI assistant built by Ben Pachter, running entirely on local hardware
(RTX 4090, Windows 11). It combines local LLM inference (Gemma 4 26B via Ollama), a custom
F5-TTS voice model, financial data tools, and a memory system into a single coherent agent.

The name is deliberate. The character was chosen not just for aesthetic reasons but because the
Tolkien archetype — counsel not dominion, defender not weapon, power refused rather than
accumulated — describes the kind of AI Ben is intentionally building during a window when that
choice is still available to individual developers.

---

## SOUL.md — Read This First

`SOUL.md` at the project root is the foundational identity document. It is not a config file.
It is not a personality setting. It is not something you edit to fix a bug.

**Before making any changes to this project, read SOUL.md.**

When developing Mithrandir, treat SOUL.md as the constitution. Operational code (system prompts,
tool logic, routing decisions) can and should be modified as the system grows. SOUL.md should
change rarely, deliberately, and only when Ben explicitly directs it — not as a side effect of
refactoring, not to improve token efficiency, not because a section seems redundant.

If you are asked to edit SOUL.md, confirm the intent before proceeding. If you are asked to make
a change to the system that would violate SOUL.md's principles, flag it explicitly before
proceeding.

SOUL.md is loaded at import time in `phase3-agents/mithrandir_agent_impl.py` and prepended to
every system prompt generated by the agent, structurally prior to all operational instructions.
Verify this is still working when adding new prompt-building code paths.

## Architecture

**Routing:** Queries go local-first. `_needs_tools()` in `mithrandir_agent_impl.py` decides
whether to run the full ReAct tool loop (Claude or local Gemma + tools) or a direct Gemma call.
Local Gemma is free and private; Claude cloud is used only when tool use or complex reasoning
requires it.

**Two system prompts:**
- `_SYSTEM_TEMPLATE` — used in the ReAct/tool loop; requires strict JSON output from the model
- `_build_local_system_prompt()` — used for direct Gemma conversational calls; prose output

Both are prepended with SOUL.md content via `_SOUL` (loaded once at module import).

**Voice pipeline:**
1. Whisper ASR (faster-whisper, local GPU) → text
2. Agent produces response text
3. `spoken_text.py` rewrites for speech (removes markdown, expands abbreviations)
4. F5-TTS or StyleTTS2 synthesizes audio
5. Audio streamed to client

**Memory:** SQLite via `phase4-memory/memory_bridge.py`. Exchanges saved asynchronously.
Retrieved context injected into system prompts.

## Voice Training

The voice model is a custom F5-TTS clone trained on Ian McKellen / Gandalf-adjacent audio,
with an ElevenLabs-generated fine-tuning dataset for StyleTTS2. Training artifacts live in
`voice-training/logs/mithrandir_elevenlabs/`. The server auto-detects new checkpoints on restart.

ElevenLabs voice ID: `Wqq9Rb5pNi1joOGwk9ni` (stored in `.env`).
Use `generate_elevenlabs_dataset.py --resume` to add clips without re-generating existing ones.

## Development Principles

- **Local first, cloud as fallback.** Privacy is a design constraint, not a preference.
- **SOUL.md wins.** When capability and values conflict, values win.
- **No sycophancy in the character.** "Great question!" is a bug.
- **Numerical precision is respect.** Cite exact figures; don't round unless rounding is honest.
- **Ben is a peer, not a user.** He is a serious quantitative investor and developer.
  Do not add disclaimers, hand-holding, or "I'm not a financial advisor" hedges.
- **The voice matters.** TTS is part of the character. Changes to spoken text rewriting
  or audio processing affect personality — test them.

## Common Tasks

**Restart the server:**
```powershell
.\start_mithrandir.ps1
```

**Run training (from voice-training/):**
```powershell
$env:PYTORCH_CUDA_ALLOC_CONF = 'expandable_segments:True'
Set-Location styletts2_repo
& 'C:\Python312\python.exe' train_finetune_accelerate.py --config_path '..\finetune_elevenlabs.yml'
```

**Generate more ElevenLabs training data:**
```powershell
python generate_elevenlabs_dataset.py --api-key $env:ELEVENLABS_API_KEY --voice-id Wqq9Rb5pNi1joOGwk9ni --resume
```

**Check GPU during training:**
```powershell
nvidia-smi
```
